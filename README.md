# frame2


## Описание проекта

Данный проект представляет собой модульное приложение на C#. Приложение
построено таким образом, что его функциональность можно расширять с
помощью подключаемых модулей без изменения ядра системы.

Основная цель проекта --- продемонстрировать: - модульную архитектуру -
внедрение зависимостей (Dependency Injection) - автоматическую загрузку
модулей - управление зависимостями между модулями

Ядро приложения не знает о конкретных реализациях модулей и работает
только через общий контракт.

------------------------------------------------------------------------

# Контракт модуля

Каждый модуль реализует интерфейс:

``` csharp
public interface IModule
{
    string Name { get; }
    List<string> Dependencies { get; }

    void RegisterServices(IServiceCollection services);
    void Initialize(IServiceProvider serviceProvider);
}
```

Модуль обязан: - иметь имя - указать зависимости - зарегистрировать
сервисы - выполнить инициализацию

# Реализованные модули

## ValidationModule

Отвечает за инициализацию правил проверки данных.

Функции: - регистрация сервиса IDataService - инициализация правил
проверки

------------------------------------------------------------------------

## ReportingModule

Формирует отчёт по данным студентов.

Зависит от:

ValidationModule

Выводит список студентов в консоль.

------------------------------------------------------------------------

## ExportModule

Экспортирует отчёт в файл.

Зависит от:

ReportingModule

Создаёт файл `report.txt`.

------------------------------------------------------------------------

# Контейнер внедрения зависимостей

Приложение использует встроенный контейнер Microsoft Dependency
Injection.

Пример регистрации:

``` csharp
services.AddSingleton<IDataService, DataService>();
```

Контейнер создаёт зависимости автоматически.

------------------------------------------------------------------------

# Проверка зависимостей

Система автоматически проверяет:

-   отсутствие требуемого модуля
-   циклические зависимости

### Пример отсутствующего модуля

    Module Reporting requires missing module Validation

### Пример циклической зависимости

    Cycle detected involving module A

------------------------------------------------------------------------

# Проверки

В проекте реализованы тесты:

1.  Проверка правильного порядка запуска модулей

# Почему добавление нового модуля не требует изменения ядра

Ядро работает только через интерфейс `IModule`.

Чтобы добавить новый модуль необходимо: 1. создать класс, реализующий
`IModule` 2. указать зависимости 3. добавить модуль в конфигурацию

Логика ядра остаётся неизменной.
